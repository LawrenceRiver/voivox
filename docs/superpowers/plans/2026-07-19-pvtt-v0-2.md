# PVTT V0.2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Turn Voice Vac into a Private Video Transcription Tunnel with one shared machine language across MCP Monitor, macOS App, and Chrome Extension, then expose a direct `transcribe_active_video` MCP flow with live capture and an accelerated-mode boundary.

**Architecture:** Keep Voice Vac as the product name and PVTT/D Channel as the technical model. `packages/core` owns transcript/session contracts, mode selection, and service state; `packages/ui` owns shared visual tokens, state labels, and the responsive tunnel view; the MCP server owns Agent-facing commands; the App owns the local bridge and full-size monitor; the Extension owns the user-authorized tab capture and in-page floating tunnel. The first production path is Live Tunnel Mode; Accelerated Decode Mode is added behind the same provider interface and falls back automatically.

**Tech Stack:** TypeScript, Vitest, React, Electron, Chrome Manifest V3, `chrome.tabCapture`, Offscreen Document, existing loopback bridge, existing local ASR runtime, FFmpeg where available, Qwen3-ASR provider adapter.

## Global Constraints

- Capture only the user-authorized target browser tab; do not capture microphone, system output, Logic Pro, Spotify, or other tabs.
- Do not modify macOS audio devices and do not bypass DRM, authentication, or site permissions.
- MCP returns structured transcript data directly; clipboard is an App/Extension convenience only.
- `processing_mode` is always explicit: `live_tunnel` or `accelerated_batch`.
- Fast Transcript must not wait for optional forced alignment; refined timestamps are a second-stage result.
- Chinese and English UI copy share the same keys and state semantics.
- No production feature is added before a failing test demonstrates its intended behavior.

- Modify: `README.md`

**Interfaces:**
- `selectProcessingMode({ requested, mediaAccessible, forceLive }): ModeDecision` selects accelerated only when bytes are legally accessible; all other cases select live.
- `AcceleratedTranscriber.transcribe(source, options)` extracts 16 kHz mono PCM, chunks with overlap, runs `ASRProvider.transcribe_batch`, and merges segments in source order.
- `ASRProvider` exposes `transcribe_batch`, `transcribe_stream`, and optional `align`; Qwen3-ASR 0.6B is the initial fast provider and 1.7B is the quality provider.
- Alignment never blocks the Stage 1 plain-text result and operates on intervals no longer than five minutes.
